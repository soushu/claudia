**Why:** 2026-03-16のセキュリティ監査で発見。Critical/一部Highは対応済み、以下は公開後に対応

## 対応予定

### High
- **APIキーのlocalStorage保存リスク** — XSS時に漏洩。CSPで緩和済みだが、理想はサーバーサイド暗号化保存。長期的にはWeb Crypto APIでの暗号化 or サーバーサイドプロキシ方式
- **パスワード強度チェック強化** — 現在8文字以上のみ。12文字以上 + zxcvbnライブラリで弱パスワード検出を推奨
- **プロンプトインジェクション対策** — システムプロンプトの長さ制限・禁止パターンフィルタ追加

### Medium
- **ログインレート制限強化** — 現在5/min → 3/minに下げ、per-emailのアカウントロック（5回失敗で15分ロック）実装
- **セッションタイトルのPydanticバリデーション** — `Field(max_length=60)` 追加（現在はサーバーで黙って切り捨て）
- **APIキーフォーマット検証強化** — Google: 30文字以上、Anthropic/OpenAI: 20文字以上に引き上げ
- **セッショントークンローテーション** — NextAuthでmaxAge/updateAge設定、機密操作時にトークン再生成

### Low
- **Expect-CTヘッダー追加** — Certificate Transparency強制
