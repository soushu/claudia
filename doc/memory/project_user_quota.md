description: ユーザーごとに無料枠のAPI使用量をカウントして制限する機能。ローンチ後にユーザー数・使用量を見て判断
type: project

ユーザー単位でGemini無料枠の使用量をカウントし、一定量で「無料枠を使い切りました」と表示する機能。

**Why:** フリーキープール（5キー）を全ユーザーで共有しているため、特定ユーザーの大量利用で他ユーザーに影響が出る可能性がある

**How to apply:** ローンチ後に実際のユーザー数と使用パターンを見てから実装判断。今はフリープールの429エラーが自然な制限として機能する

**実装案:**
- messagesテーブルの当日カウントで十分（専用テーブル不要）
- 制限値例: ユーザーあたり1日50リクエスト（フリープール総RPD 2,500 ÷ 想定ユーザー50人）
- リクエスト前にチェック → 超過時はストリーム開始前にエラー返却
